1. What is DOM and what is the purpose of DOM?

The Document Object Model (DOM) is a programming interface for web documents. It is a document programming interface that offers a structured model of the HTML, XHTML, or XML document in the form of a tree. Using computer languages like JavaScript, web developers can interact with and modify the content and structure of a web page using the DOM.

The DOM was created to offer a standardized method for representing and modifying web page content. It enables web designers to dynamically alter a web page's content without having to reload the whole thing.

  • Modify an HTML element's text content.
  • Dynamically add or remove HTML components.
  • An HTML element's style or properties can be modified.
  • User input and events, including button clicks or form submissions, are responded to.

2. How will you select HTML elements using DOM? Name the DOM methods.

JavaScript is most commonly used to get or change the content or values ​​of HTML elements on the page, and to apply effects such as fade-ins, fade-outs, animations, etc. JavaScript's DOM Selectors are used to choose specific HTML elements from a document. There are five ways to use selectors to choose elements in a DOM.

  • getElementsByTagName()
  • getElementsByClassName()
  • getElementById()
  • querySelector()
  • querySelectorAll()

What is event bubble?

Event bubbling is a phenomenon in which an event that is triggered on a particular element is propagated up through its parent elements, all the way up to the document object. The click event will first be triggered on the button element, then on the div element, then on the section element, and so on, until it reaches the document object. If the event bubbles up to a parent element that has its own event listener for the same event type, that event listener will also be triggered. To prevent event bubbling, we can use the event.stopPropagation() method inside the event listener for the innermost element. This will stop the event from propagating up to its parent elements, ensuring that only the intended element's event listener is triggered.

What is a callback function and why will you use it?

A callback function is a function that is passed as an argument to another function and is executed after some operation is completed. The callback function is executed once the response is received, allowing the application to process the data. The main benefit of using callback functions is that it allows for non-blocking execution of code. By using a callback function, the long-running operation can be executed asynchronously in the background, allowing the main thread to continue running and keeping the application responsive.
